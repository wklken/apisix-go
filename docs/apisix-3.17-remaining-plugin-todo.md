# APISIX 3.17 Remaining Plugin TODO Plan

This is the remaining APISIX 3.17 parity gap catalog. Normal Go-scope implementation work is complete for the current target; entries below record explicit runtime/native limitations, separate-subsystem prerequisites, or monitor-only conditions rather than pretending those differences are implemented.

OpenResty-native, NGINX-native, Lua-runtime-native, serverless function plugins, and missing/deferred official defaults are not normal TODOs. Not required unless explicitly requested: `ext-plugin-pre-req`, `ext-plugin-post-req`, `ext-plugin-post-resp`, `inspect`, `serverless-pre-function`, `serverless-post-function`, OCSP stapling internals, exact OpenResty phase timing, `ngx_lua` APIs, Lua code execution, NGINX buffering internals, and shared-dict/lrucache exactness. Do not create normal plugin implementation tasks for these native/runtime-only items.

Use this file as the planning backlog. Use `README.md` and `docs/apisix-3.17-plugin-parity-checklist.md` as the status surfaces after each implemented slice.

## Suggested Implementation Order

The OAS validator also accepts free-form exploded `form` query objects backed by `additionalProperties`; duplicate scalar properties remain rejected. It now supports bounded Parameter Object `content` serialization for one JSON/`+json` or `text/plain` media type with schema validation; malformed JSON, duplicate values, and unsupported parameter codecs are rejected explicitly.

1. Keep the shared Go passive health-aware upstream/load-balancer abstraction as the owner for observed HTTP/TCP outcomes; active probes and the `/v1/healthcheck` state API remain explicit deferments rather than plugin-local fakes.
2. Keep the official Kafka PubSub owner and TLS/SASL/PLAIN boundary stable; external broker smoke coverage and mechanisms beyond PLAIN remain later work.
3. Revisit gRPC annotation/hooks/streaming only after the Go route/proxy layer exposes an explicit protobuf streaming binding and cancellation contract.
4. Fix `body-transformer`, `data-mask`, `oas-validator`, `traffic-split`, `response-rewrite`, `fault-injection`, or `uri-blocker` only when a concrete APISIX-vs-Go mismatch is reproduced; the current normal request/validation slices are already covered by focused and route-chain tests.
5. Keep OpenResty/NGINX/Lua-native behavior and the four missing external-runtime defaults documented as deferred rather than adding placeholders.
